## 2. How do the four pillars of OOP—Inheritance, Polymorphism, Abstraction, and Encapsulation—help manage logic and reduce complexity in large-scale TypeScript projects?

## Introduction
 typescript based project almost solves the type safety related Issues, but the main problem of a project is its complexity, like as codebase grows logic and featuring becomes a headache, that's why the ultimate solution is OOP (Object Oriented Programming). OOP provides a blueprint to the project so that we can reduce complexity and organize our codebase. The four pillar of OOP are 

 - Inheritance The First Pillar of OOP
 - Polymorphism The Second Pillar of OOP
 - Abstraction The Third Pillar of OOP
 - Encapsulation The Fourth Pillar of OOP

## Body

 - Inheritance

Inheritance is a fundamental of OOP mechanism where a new class (child) derives properties from a Existing class(parent), the benefit of using inheritance is make code more efficient and organized, inheritance increases code reusability , provide hierarchical organization.

 - Polymorphism

Like Inheritance polymorphism also a OOP fundamental concept that refers to the ability of a single interface or method to handle different underlying forms. It mainly helps in Scalability, Maintainability and reusability of code.

 - Abstraction

In OOP the most interesting and important fundamental concept is abstraction, the think of abstraction is like you have an idea what to do but there is no clean implementation.

 - Encapsulation

The fourth pillar of OOP is Encapsulation, it means keeping the data and methods safe and private and gave safe access through method, there are 3 modifier for encapsulation which are , public , private and protected


## Conclusion

The four pillar of OOP is really important when we are working on a complex project it help us to maintain our code base increase better readability , reusability , ignore conflicts and facilitate developers.
